# TAPD 需求描述模板

本文档定义了创建 TAPD 需求单据时使用的描述模板。

---

## 父需求描述模板

```markdown
## 需求背景
<从 PRD「1.1 背景说明」提取>

## 目标用户
<从 PRD「1.2 目标用户」提取>

## 核心价值
<从 PRD「1.3 核心价值」提取>

## 功能模块
| 模块 | 优先级 | 说明 |
|------|--------|------|
| <模块1> | P0 | <概述> |
| <模块2> | P1 | <概述> |

## 相关文档
| 文档类型 | 链接 |
|---------|------|
| 📄 需求文档 | [查看](<iWiki PRD 链接>) |
| 🎨 交互原型 | [查看](<iWiki 原型链接>) |
| ✅ 审核报告 | [查看](<iWiki 报告链接>) |
```

### 字段说明

| 字段 | 取值来源 |
|------|---------|
| 需求背景 | PRD「一、需求背景 > 1.1 背景说明」 |
| 目标用户 | PRD「1.2 目标用户」 |
| 核心价值 | PRD「1.3 核心价值」 |
| 功能模块表 | PRD「二、功能优先级」或「三、功能详细设计」章节标题 |
| 相关文档 | 上传到 iWiki 后生成的链接 |

## 子需求描述模板

```markdown
## 功能概述
<从 PRD 对应模块的「功能概述」提取>

## 功能说明
- 菜单路径：<路径>
- 页面 URL：<URL>

## 字段说明
<从 PRD 对应模块的「字段说明」表格提取>

## 核心交互
<从 PRD 对应模块的「交互说明」表格提取>

## 相关文档
- [查看 PRD 完整描述](<iWiki PRD 链接>#3.x-模块名)

## 验收标准
1. <根据功能概述推导的验收点>
2. <根据交互说明推导的验收点>
```

### 字段说明

| 字段 | 取值来源 |
|------|---------|
| 功能概述 | PRD 对应 `### 3.x` 章节的「功能概述」 |
| 菜单路径 | PRD 对应章节的「功能说明」中的路径 |
| 页面 URL | PRD 对应章节的「功能说明」中的 URL |
| 字段说明 | PRD 对应章节的「字段说明」表格 |
| 核心交互 | PRD 对应章节的「交互说明」表格 |
| 验收标准 | 根据功能概述和交互说明推导 |

### 锚点链接格式

iWiki 锚点链接格式：`https://iwiki.woa.com/pages/viewpage.action?pageId={docid}#章节标题`

示例：`https://iwiki.woa.com/pages/viewpage.action?pageId=123456#3.1-集群列表`

## 评论模板

创建完成后，在父需求上添加评论：

```markdown
## 需求文档归档

### iWiki 文档
| 文档 | 链接 |
|------|------|
| 需求文档 | [查看](<PRD 链接>) |
| 交互原型 | [查看](<原型链接>) |
| 审核报告 | [查看](<报告链接>) |

### 子需求清单
| 序号 | 名称 | 优先级 | Story ID |
|------|------|--------|----------|
| 1 | <模块A> | P0 | <story_id> |
| 2 | <模块B> | P1 | <story_id> |

### 创建信息
- 创建时间：<当前时间>
- 数据来源：PRD 自动导入
```

### 评论用途

1. 记录本次需求的所有关联文档链接
2. 汇总子需求清单，方便查看需求拆分情况
3. 记录创建时间和数据来源，便于追溯
