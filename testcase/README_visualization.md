# BKLog 测试报告可视化页面使用说明

## 概述

`test_report_visualization.html` 是一个美观的可视化测试报告页面，用于展示BKLog功能测试的详细结果和截图。

## 功能特性

- 📊 **测试概览**：统计卡片和表格展示测试总体情况
- 📋 **详细结果**：每个测试用例的完整测试信息
- 📸 **截图展示**：支持多张截图展示，响应式布局
- 🎨 **美观设计**：现代化UI设计，渐变色彩，响应式布局
- ✅ **状态标识**：清晰的状态标识（通过/部分通过/失败）

## 使用方法

### 1. 打开页面

直接在浏览器中打开 `test_report_visualization.html` 文件：

```bash
# macOS
open test_report_visualization.html

# Linux
xdg-open test_report_visualization.html

# Windows
start test_report_visualization.html
```

或者使用本地服务器：

```bash
# Python 3
python3 -m http.server 8000

# Node.js (需要安装 http-server)
npx http-server -p 8000
```

然后在浏览器中访问：`http://localhost:8000/test_report_visualization.html`

### 2. 添加截图

1. 将测试截图保存到 `screenshots/` 目录
2. 使用规范的文件名（参考 `screenshots/README.md`）
3. 刷新页面即可看到截图

### 3. 截图命名规范

- 格式：`test_case_{用例编号}_{功能描述}.png`
- 示例：
  - `test_case_1_highlight.png`
  - `test_case_2_tooltip.png`

## 页面结构

### 1. 页面头部
- 测试报告标题
- 测试日期、环境、执行者等元信息

### 2. 测试概览
- **统计卡片**：总测试用例数、已执行、通过、部分通过、通过率
- **测试用例表格**：所有测试用例的执行状态和结果

### 3. 详细测试结果
每个测试用例包含：
- 测试目标
- 测试步骤（带状态标识）
- 实际结果
- 测试截图（支持多张）
- 测试状态

### 4. 测试总结
- 总体统计信息
- 总体评价

## 自定义修改

### 修改测试数据

编辑HTML文件中的相应部分：

1. **统计卡片**：修改 `.stats-grid` 中的数字
2. **测试用例表格**：修改 `<tbody>` 中的表格行
3. **详细结果**：修改 `.test-case` 部分的内容

### 修改样式

编辑 `<style>` 标签中的CSS：

- **颜色主题**：修改渐变色彩 `#667eea` 和 `#764ba2`
- **布局**：调整 `grid-template-columns` 等布局属性
- **字体**：修改 `font-family` 和 `font-size`

### 添加新测试用例

1. 复制一个 `.test-case` 块
2. 修改测试用例编号和内容
3. 添加对应的截图（如果需要）

## 截图要求

- **格式**：PNG（推荐）或 JPG
- **分辨率**：建议宽度至少 1200px
- **文件大小**：建议每个截图不超过 2MB
- **内容**：清晰展示测试的关键步骤和结果

## 浏览器兼容性

- ✅ Chrome/Edge (最新版本)
- ✅ Firefox (最新版本)
- ✅ Safari (最新版本)
- ⚠️ IE 11（部分功能可能不支持）

## 注意事项

1. **截图路径**：确保截图文件路径正确，相对于HTML文件
2. **文件大小**：如果截图文件过大，可能影响页面加载速度
3. **响应式设计**：页面支持移动端查看，但建议在桌面端查看以获得最佳体验
