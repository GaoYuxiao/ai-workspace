# 设计需求转文档

## 核心能力

基于用户提供的各种设计资料和需求信息，自动生成完整的产品文档体系，包括交互设计、功能规范、架构设计等多维度文档。

## 支持的输入类型

### **设计稿输入**
- **移动端设计稿** - iOS/Android 应用界面设计
- **Web端设计稿** - 网站、后台管理系统界面
- **原型图** - 低保真/高保真原型设计
- **线框图** - 产品结构和布局设计

### **截图资料**
- **界面截图** - 现有产品界面截图
- **竞品分析截图** - 参考产品功能截图
- **用户流程截图** - 操作流程演示截图

### **Figma设计稿**
- **Figma链接** - 通过MCP工具拉取设计稿数据
- **设计组件** - 组件库和设计系统
- **设计规范** - 颜色、字体、间距等规范

### **需求描述文本**
- **PRD文档** - 产品需求文档
- **功能需求** - 具体功能点描述
- **用户故事** - 用户场景和需求描述
- **业务目标** - 产品目标和价值主张

## 生成文档类型

### 【产品交互文档】
详细描述产品的交互设计规范和用户体验流程。

#### 主要内容
- **导航设计** - 主导航、次导航（按需）
- **页面结构** - 产品功能在页面区域上的呈现
- **交互流程设计** - 组件之间的交互流程，需跟用户确认

#### 输出格式
Markdown文档格式

### 【产品功能文档】
全面描述产品的功能模块和业务逻辑，专注于核心功能详述。

#### 主要内容
- **功能描述** - 每个功能模块的详细功能说明
- **使用场景** - 功能的应用场景和用户需求
- **操作流程** - 具体的操作步骤和流程
- **业务规则** - 功能相关的业务逻辑和规则
- **异常处理** - 错误情况的处理方式

#### 输出格式
Markdown文档格式

### 【功能结构图】
可视化展示产品的功能模块结构和层次关系。

#### 主要内容
- **功能树状图** - 展示功能的层级结构（父级到子级按从左到右顺序）
- **模块关系图** - 展示模块间的依赖关系
- **用户角色图** - 不同用户角色的功能权限
- **页面结构图** - 页面层级和导航结构

#### 输出格式
- Mermaid图表代码
- HTML可视化页面
- PNG图片格式（同级节点必须对齐，不同节点间不允许重叠）

### 【系统架构图】
展示产品的技术架构和系统设计。

#### 主要内容
- **整体架构图** - 系统的宏观架构设计
- **技术栈图** - 前端、后端、数据库技术选型
- **部署架构图** - 服务器、CDN、负载均衡等部署方案
- **数据流图** - 数据在系统中的流转路径
- **安全架构图** - 安全防护和权限控制机制

#### 输出格式
- Mermaid架构图
- 系统架构说明文档
- PNG架构图片

### 【使用流程图】
描述用户使用产品的完整流程和操作路径。

#### 主要内容
- **示例功能1** - 需跟用户确认
- **示例功能2** - 需跟用户确认
- **示例功能3** - 需跟用户确认

#### 输出格式
- 流程图Mermaid代码
- 流程说明文档
- PNG流程图片（不同节点间不允许重叠）

### 【UML序列图】
展示系统各组件间的交互时序和消息传递。

#### 主要内容
- **用户交互序列** - 用户与系统的交互时序
- **系统内部序列** - 系统组件间的调用关系
- **API调用序列** - 前后端API交互时序
- **数据处理序列** - 数据处理的完整流程
- **异常处理序列** - 错误情况的处理时序

#### 输出格式
- PlantUML代码
- Mermaid序列图
- PNG序列图片

## 工作流程

### 第一阶段：需求分析
1. **输入解析** - 分析用户提供的设计稿、截图或文本描述
2. **需求提取** - 识别核心功能、用户场景、业务目标
3. **信息整理** - 整理和结构化分析结果
4. **需求确认** - 与用户确认理解的准确性

### 第二阶段：文档生成
1. **产品交互文档** - 基于设计稿生成交互规范
2. **产品功能文档** - 基于需求生成功能说明
3. **功能结构图** - 生成功能模块结构图
4. **系统架构图** - 设计技术架构方案
5. **使用流程图** - 绘制用户操作流程
6. **UML序列图** - 生成系统交互时序图

### 第三阶段：优化完善
1. **文档审查** - 检查文档的完整性和一致性
2. **格式优化** - 统一文档格式和样式
3. **交叉引用** - 建立文档间的关联关系
4. **质量提升** - 优化文档的可读性和专业性

## 使用指南

### 快速开始

```markdown
# 使用示例

## 输入设计稿
请提供您的设计稿（图片、Figma链接或需求描述），我将为您生成完整的产品文档。

## 指定文档类型
您可以指定需要生成的文档类型：
- 产品交互文档
- 产品功能文档  
- 功能结构图
- 系统架构图
- 使用流程图
- UML序列图

## 自定义需求
如有特殊要求，请详细说明：
- 目标用户群体
- 技术栈偏好
- 业务场景特点
- 性能要求等
```

### 最佳实践

1. **提供清晰的设计稿** - 确保图片清晰，文字可读，如有主/次导航，需确认需求的页面处于哪一级产品结构下
2. **详细描述需求** - 基于第1项确定的产品结构，提供该结构的业务场景和用法介绍
3. **明确目标用户** - 说明产品的目标用户群体
4. **指定技术要求** - 如有技术栈偏好，请提前说明
5. **迭代优化** - 根据生成结果提供反馈，持续优化

### 输出规范

- **文件组织** - 所有文档按类型分类存放
- **关联引用** - 文档间建立清晰的引用关系
- **图表标准** - 统一的图表样式和标注规范
- **命名规范** - 规范的文件和章节命名
- **格式统一** - 一致的Markdown格式和样式

## 技术实现

### 图表生成工具
- **Mermaid.js** - 流程图、架构图、序列图
- **PlantUML** - UML图表和系统设计图
- **Python PIL** - 自定义图表和可视化
- **HTML/CSS** - 交互式图表和文档

### 文档处理
- **Markdown** - 主要文档格式
- **HTML** - 富文本展示和交互
- **PNG/SVG** - 图片输出格式
- **PDF** - 最终文档输出

### 质量保证
- **模板标准化** - 使用标准化的文档模板
- **内容校验** - 自动检查文档完整性
- **格式规范** - 统一的格式和样式标准
- **版本管理** - 支持文档版本控制和更新
