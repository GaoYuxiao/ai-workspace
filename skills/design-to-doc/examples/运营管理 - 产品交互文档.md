# 智能体运营管理 产品交互文档

## 1. 导航设计

### 1.1 主导航
#### 主导航设计
**定义**：位于智能体详情页顶部的Tab导航，运营管理作为其中一个Tab页面。

**设计要求**：
- **位置布局** - 位于智能体详情页顶部，与现有Tab（配置、发布、使用渠道、使用文档、使用权限）并列
- **菜单结构** - 运营管理位于使用渠道Tab之后，使用文档Tab之前
- **视觉样式** - 保持与现有Tab一致的样式风格，使用平台标准的Tab组件
- **交互状态** - 支持默认、悬停、激活状态，激活时显示下划线或背景色变化

### 1.2 次导航
#### 次导航设计
**定义**：运营管理内的分组导航，按智能体类型进行分组展示，无需额外的筛选控件。

**设计要求**：
- **位置布局** - 位于运营管理内容区，按智能体类型垂直分组
- **菜单结构** - 分为两大类：单智能体运营数据、流程智能体运营数据
- **视觉样式** - 使用分组标题和卡片布局，保持统一的组件风格
- **交互状态** - 支持卡片悬停效果，点击展开详情

## 2. 页面结构

### 2.1 单智能体数据展示区
**功能定位**：展示单智能体在不同渠道下的会话运营数据。

**数据字段**：
- **记录ID** - 显示会话的唯一标识号，支持点击查看详情
- **记录标题** - 显示会话的标题或主题，支持文本截断和悬停显示完整内容
- **来源渠道** - 显示会话来源渠道
  - 调试态：小鲸渠道
  - 正式态：API渠道、小鲸渠道、蓝鲸插件渠道
- **状态** - 显示记录执行状态
  - 已完成（绿色圆点）
  - 进行中（蓝色圆点）
  - 已失败（红色圆点）
- **执行开始** - 显示记录开始执行的时间，格式：YYYY-MM-DD HH:mm:ss
- **执行结束** - 显示记录执行完成的时间，格式：YYYY-MM-DD HH:mm:ss
- **执行人** - 显示记录的执行人员信息

### 2.2 流程智能体数据展示区
**功能定位**：展示流程智能体在不同渠道下的任务执行数据。

**数据字段**：
- **记录ID** - 显示任务的唯一标识号，支持点击查看详情
- **记录名称** - 显示任务的完整名称，支持文本截断和悬停显示完整内容
- **来源渠道** - 显示任务来源渠道
  - 调试态：小鲸渠道、任务调用渠道
  - 正式态：任务调用渠道、API渠道、小鲸渠道、蓝鲸插件渠道
- **状态** - 显示记录执行状态
  - 已完成（绿色圆点）
  - 进行中（蓝色圆点）
  - 已中断（灰色圆点）
  - 已失败（红色圆点）
- **执行开始** - 显示记录开始执行的时间，格式：YYYY-MM-DD HH:mm:ss
- **执行结束** - 显示记录执行完成的时间，格式：YYYY-MM-DD HH:mm:ss
- **执行人** - 显示记录的执行人员信息

### 2.3 详情面板区
**功能定位**：展示选中记录的完整详细信息。

**内容结构**：
- 面板头部：显示标题和关闭按钮
- 面板主体：显示详细信息，支持滚动
- 面板底部：显示操作按钮（如有）

## 3. 交互流程设计

### 3.1 用户进入流程
**流程描述**：用户从智能体详情页进入运营管理功能的完整流程。

**关键步骤**：
1. 用户访问智能体详情页
2. 点击"运营管理"Tab
3. 系统加载运营数据
4. 按智能体类型分组展示数据

### 3.2 详情查看流程
**流程描述**：用户查看单条记录详细信息的流程。

**关键步骤**：
1. 在数据列表中点击记录
2. 右侧滑出详情面板
3. 查看完整信息
4. 关闭详情面板
